{Pushpop.TableViewCell} dequeueReusableCellWithIdentifier(reuseIdentifier)
Returns a new or recycled TableViewCell with the specified reuse identifier. This method will first attempt to reuse a recycled TableViewCell with the specified reuse identifier. If no recycled TableViewCells with that reuse identifier are available, a new one will be instantiated and returned. The TableViewCell that is returned is always added to the |visibleCells| array.
Parameters:
reuseIdentifier
deselectAllRows()
De-selects all rows in the table.
deselectRowAtIndex(index, animated)
De-selects the row at the specified index and optionally animates the de-selection if the row is currently visible. NOTE: This method will not modify any other existing selections.
Parameters:
{Number} index
The index of a row in the data source to de-select.
{Boolean} animated Optional
A flag indicating if the de-selection should be animated if the row is currently visible.
{Number} getCalculatedNumberOfVisibleCells()
Returns the calculated number of how many cells should currently be visible. NOTE: There may be moments in time where this calculated number differs from the number of cells in the array returned from getVisibleCells() (e.g.: Immediately following an orientation change or window resize).
{Pushpop.TableViewDataSource} getDataSource()
Returns the TableViewDataSource for this TableView.
{Boolean} getEditing()
Determines if this TableView is in editing mode.
{Array} getIndexesForSelectedRows()
Returns the indexes in for the selected rows. NOTE: The array contains indexes of rows in the data source, NOT indexes of cells in the DOM. If no rows are selected, this array will be empty.
{Number} getIndexForSelectedRow()
Returns the index in for the first selected row. NOTE: This is an index of a row in the data source, NOT an index of a cell in the DOM. If no rows are selected, this method will return -1.

{Object} getReusableCells()
Returns an object containing arrays of reusable cells that are not currently visible. The object contains an array property for each "reuse identifier" in use. For example, if there are two types of TableViewCells in this TableView with two different reuse identifiers (cellType1 and cellType2), this object would contain two arrays that can be accessed using square bracket notation like this: reusableCells['cellType1'] or reusableCells['cellType2']
