{Pushpop.TableViewDataSource} getChildDataSourceForRowAtIndex(index)
Creates and returns a new data source for the row at the specified index if the item at that index contains a child data source as determined by the rowHadChildDataSourceAtIndex() method. NOTE: This is the default implementation and should be overridden for data sources that are not driven directly from an in-memory data set. In the default implementation, the |dataSourceKey| that is set using the setDataSourceKey() method is used to retrieve the array of objects for that key on the item at the specified index. The array of child objects are then used to create a new data source. The new data source is automatically given the same child data source key in order to continue chaining nested data n-levels deep.
Parameters:
{Number} index
The index of the row to retrieve a child data source for.
{String} getChildDataSourceKey()
Returns a string that specifies a key on this data source's objects that may contain an array of data for a child data source.
{Array} getDataSet()

{String} getDefaultReuseIdentifier()
Returns the default reuse identifier that this data source will use when a reuse identifier is not specified for a particular item.
{Array} getFilteredDataSet()
Returns the filtered in-memory data set this data source will provide to the table view. NOTE: This may not be utilized by a custom data source.
{Object} getFilteredItemAtIndex(index)
Returns the item at the specified index of the filtered data set contained within this data source. NOTE: This method is called directly by the table view's rendering process. It should yield the same data that is used by the getCellForRowAtIndex method in most cases. IMPORTANT: When working with a data source that is driven from an in-memory data set, this method should ALWAYS be used to access elements from the filtered data set. It is NOT RECOMMENDED that the elements be accessed using "[ ]" notation on the filtered data set's array directly.
Parameters:

IMPORTANT: When working with a data source that is driven from an in-memory data set, this method should ALWAYS be used to access elements from the complete data set. Returns the number of filtered items contained within this data source. NOTE: This method is called directly by the table view's rendering process. setFilterFunction(filterFunction)
Sets a filter function to be used when searching this TableView. The filter function gets called for each item in the data set during a search. A valid filter function must take two parameters (regExp, item) and return a Boolean value. The |regExp| parameter contains a RegExp object based on the search string to be used to match items in the data set. The |item| parameter contains an item from the data set that the search string in the RegExp should be tested against. The provided filter function should return a Boolean value: |true| if the item should match the search string or |false| if it should be filtered out. NOTE: This may not be utilized by a custom data source.
Parameters:
{Function} filterFunction
The filter function to be used when searching this TableView.

Sets the TableView this data source should be bound to.
Parameters:
{Pushpop.TableView} tableView
The TableView to bind this data source to.
{Boolean} shouldReloadTableForSearchString(searchString, isCaseSensitive)
Determines if the table should be reloaded following a change in the search string. The default implementation assumes that the data set is fully loaded into memory and executes the current filter function against each item in the data set. If the filtered data set has changed since the last reload, it will return |true| which will force the associated TableViewSearchBar to reload the data for the TableView. In a custom data source that does not use an in-memory data set (e.g.: WebSQL or HTML5 LocalStorage), it is recommended to override this method to perform any necessary queries asynchronously and immediately return |false|. Once the asynchronous queries have completed, the application should then manually call .reloadData() on the TableView to force an update (See the WebSQL demo for an example on this implementation).
Parameters:
{String} searchString
The search string to be used for matching items in the data set.
{Boolean} isCaseSensitive Optional
An optional boolean flag for forcing a case-sensitive search.
